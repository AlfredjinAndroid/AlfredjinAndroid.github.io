前端可编辑select标签仿搜索操作

发表于 2017-08-19 | 分类于 j2e
由于一些需要,咱就这么做了

整体思路:

select本身是不可编辑的，所以需要放一个可编辑的input覆盖原有的select。所以就要在编辑input的时候去触发select的事件，由于触发select的下拉事件的ActiveX等的发方法只能在IE浏览器中实用，所以实用通过改变select的size来达到相应的目的。在需要收起列表的时候也同样的设置size来实现。
在选择下拉框内容时改变input显示内容

#seval : Input标签ID

#op : select标签ID
_val : 选中的select的选项的内容

思路:

选择了下拉列表中选项后，会触发select的change事件，所以在change事件中，当选择了某选项，该选项的selected属性会触发，所以查询select下有selected的option，获取到该option的值并赋值给input

input内容改变的时候，下拉框显示，并更新显示的option

#seval : input标签ID

#op : select标签ID
ops : 存放option对象的数组
ops数组

思路:
在页面加载成功后就遍历select下的option标签，并存放到数组中备用

事件操作过程

思路:

列表下拉的思路:由于ActiveX只能在IE浏览器中使用，所以不能使用该方法。在input中有值改变的时候，就要捕捉文本，并显示下拉框，下拉列表无法手动调用下拉事件(click,focus,mouse等),所以改变了下拉列表的属性size的值，并将select置于上方(z-index=3),而input要在select上方(z-index=4)，其他css只是调整位置等属性

实时匹配思路:当input中的内容发生改变的时候，就比较input的值和select中option的值，如果option的值包含输入内容的值，那么该对象就要显示，否则隐藏该对象。

注意事项:

在设置input和select的z-index时，需要先设置input，再设置select

匹配字符串方法A.indexOf(B);A是否包含B，如果有，返回值>0，否则返回-1;

上边的方法是选项包含输入的内容就会显示，而有时候我们的需求是只匹配开头是输入内容的部分，也就是向后匹配的方式，那么就需要用下边的这种方法

当点击了其他地方的时候，需要select收起下拉列表

#seval : input标签ID

#op : select标签ID

思路:

搜索框在input没有焦点，并且选择框也不存在焦点的时候，就要收起下拉框，由于之前设置的下拉列表的显示是控制的size，所以收起时也是操作size属性

注意事项:

在开始写的时候，还能第一次成功，之后显示了下拉列表，但是无法点击下拉列表就被收起，主要原因是当input不存在焦点的时候，此时的select也不存在焦点，所以就被收起，在select失去焦点的事件前为select添加焦点，就可以像原一样点击

以上方法在IE8中是不兼容的，而且在触发事件的时候，会出现stackover flow at line 0 的错误，主要是存在堆栈溢出，解决方法：

在设置select和input时在原标签上设置css属性，在js里值设置点击后的size属性,避免了js代码臃肿问题。

而在IE8中不能直接对select下的option标签进行隐藏和显示操作，即是Jquery里的hide()和show()方法不能够使用。那么可以使用添加和移除的方法来实现这一效果。
